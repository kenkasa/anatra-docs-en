# Usage of fanatra

## Introduction

**fanatra** stands for **"Fortran ANATRA"** and, as its name suggests, is a collection of analysis programs written in Fortran. This chapter describes how to use **fanatra**. Since **fanatra** performs analyses without invoking VMD, it can also be used on computing systems where the use of VMD on compute nodes is difficult. A list of available analysis modes can be displayed by executing ``fanatra -h``.

```
$ fanatra -h
Available analysis in Fortran ANATRA
histogram               (hist)   : Histogram Analysis
potential_of_mean_force (pmf)    : Free-energy Profile Analysis
restricted_radial       (rrpmf)  : Restricted Radial Analysis
extract_frame           (extf)   : Extract Frames Using CV
random_frame            (randf)  : Randomly Extract Frames
state_define            (sdef)   : Define States From Time-series Data
moving_average          (movave) : Moving Average Analysis
average_function        (avef)   : Function Average Analysis
bootstrap_prepper       (boot)   : Bootstrap Sample Generator
return_prob             (rp)     : Returning Probability Analysis
rprate                  (rprate) : Rate Constant Analysis based on RP theory
iepdyn                  (iepdyn) : Integral-equation Formalism of Population Dynamics
```

Among the above, entries such as ``histogram`` are analysis modes, and their abbreviated names (e.g., ``hist``) can also be used.

While **anatra** performs analyses by specifying parameters directly on the command line, **fanatra** uses a separate parameter file. For example,

```
$ fanatra hist control_file
```

The parameter file must be written in the Fortran namelist format.

In addition, the format and contents of the parameter file for each analysis mode can be displayed together with a brief explanation and a sample input by executing ``fanatra <analysis_mode> -h``. Therefore, the basic usage of each analysis mode can be understood from the help message. An example for ``histogram`` is shown below.

```
$ fanatra histogram -h

==================================================

              Histogram Analysis

==================================================
&input_param
 fcv      = "cvdata1" "cvdata2" ! Time-series CV data
 flist_cv = "list" ! CV-file list (Either fcv or flist_cv should be specified)
/
&output_param
 fhead = "run" ! header of output filename
/
&option_param
  dx       = 0.1d0  ! grid spacing
  xsta     = 0.0d0  ! minimum value of the coordinate
  nx       = 100    ! number of grids
  ncol     = 1      ! number of data column
/


:::
1st column: time (arbitrary), Nth column (N>1): data
example)

0.00   0.02120
0.01   0.03140
0.02   0.22310
....   .......
```

## Common Parameters

Most parameter files in **fanatra** are based on the Fortran namelist format. Parameters are specified in the **input_param**, **output_param**, and **option_param** sections.

The **input_param** and **output_param** sections, which are common to all analysis modes, are described below.

### &input_param

This namelist specifies parameters related to input files.

**Parameter**                         | **Function**                                                      | **Example**
:-------------------------------------|:-----------------------------------------------------------------:|:------------------------------:
ftraj                                 | Specify input trajectory file(s) (multiple files allowed)         | ftraj      = "inp1.dcd", "inp2.dcd"
fcv                                   | Specify input time-series data file(s) (multiple files allowed)   | fcv        = "cv1.txt", "cv2.txt"
fweight                               | Specify statistical weight file(s) corresponding to the input time-series data (multiple files allowed) | fweight = "weight1.txt", "weight2.txt"
flist_traj$^{1}$                      | Specify a file containing a list of input trajectory files        | flist_traj = "list.txt"
flist_cv$^{2}$                        | Specify a file containing a list of input time-series data files  | flist_cv   = "list.txt"
flist_weight$^{3}$                    | Specify a file containing a list of statistical weight files      | flist_weight = "list.txt"

$^{2}$ If both ``fcv`` and ``flist_cv`` are specified, the contents of ``flist_cv`` take precedence.  
$^{3}$ If both ``fweight`` and ``flist_weight`` are specified, the contents of ``flist_weight`` take precedence.

    * Example: ``fweight = "weight1.txt", "weight2.txt"``

Statistical weights obtained from enhanced sampling methods or the Multistate Bennett Acceptance Ratio (MBAR) method can be used.     As shown in the example, multiple files can be specified.

    ```
    inp4.dcd
    ```

    * Default: N/A
    * Example: ``flist_cv = "list.txt"``

    Specifies a file containing a list of input time-series data files. The contents specified by ``flist_cv`` take precedence over those specified by ``fcv``.

    The list file must contain one input time-series data filename per line, as shown below.

    ```
    inp1.txt
    inp2.txt
    inp3.txt
    inp4.txt
    ```

 ---
* **``flist_weight``**
    * Default: N/A
    * Example: ``flist_weight = "list.txt"``

    Specifies a file containing a list of input statistical weight files. The contents specified by ``flist_weight`` take precedence over those specified by ``fweight``.

    The list file must contain one input statistical weight filename per line, as shown below.

    ```
    inp1.txt
    inp2.txt
    inp3.txt
    inp4.txt
    ```

### Format of Time-series Data

The format of the time-series data is as follows.

* 1st column: Time axis
* 2nd column: First time-series data
* 3rd column: Second time-series data

The first column is ignored in all analysis modes, and the second column is treated as the first data column.

### Format of Statistical Weight Data

The format of the statistical weight data is as follows.

* 1st column: Time axis
* 2nd column: Statistical weight

The first column is ignored in all analysis modes, and only the second column is used.

The number of rows in the statistical weight data file must be identical to that of the corresponding time-series data file.

## Histogram (``histogram``, ``hist``)

In this analysis mode, the distribution function $P\left(x\right)$ is calculated from a given time-series data set $\xi\left(t\right)$.

\begin{align}
P\left(x\right) =  \dfrac{1}{N}\Braket{\delta\left(x - \xi\right)}
\end{align}

where $N$ is the number of data points. The above expression satisfies the normalization condition

\begin{align}
\int_{-\infty}^{\infty} dx P\left(x\right) = 1
\end{align}

### &option_param

* **``dx``**
    * Default: 0.1
    * Example: ``dx = 0.1``

    Specifies the grid spacing $\Delta x$ used to calculate the distribution function.

----
* **``xsta``**
    * Default: 0.0
    * Example: ``xsta = 0.0``

    Specifies the starting position of the grid, $x_{0}$. If the position of the $i$-th grid point is denoted by $x_{i}$,

    \begin{align}
    x_{i} = x_{0} + \left(i - 1 \right) \Delta x
    \end{align}

    holds.

----
* **``ncol``**
    * Default: 1
    * Example: ``ncol = 10``

    Specifies the number of data columns contained in each time-series data file.

### Format of Output File

* ``*.dat`` (the extension can be changed by ``file_extension``)
    * 1st column: $x$ coordinate
    * 2nd column: Distribution function $P\left(x\right)$

### Example

The following example calculates the distribution function $P\left(x\right)$ in the range $5 \leq x \leq 10$ from a time-series data file ``inp.ts`` containing three data columns.

```
#!/usr/bin/env bash

cat << EOF > run.inp
&input_param
fcv = "inp.ts"
/
&output_param
fhead = "run"
/
&option_param
dx   = 0.1d0
xsta = 5.0d0
nx   = 50
ncol = 3
/
EOF

fanatra hist run.inp
```
## Potential of Mean Force (``potential_of_mean_force``, ``pmf``)

In this analysis mode, the free-energy profile $W\left(\boldsymbol{\zeta}\right)$ along the reaction coordinate $\boldsymbol{\zeta}$ is calculated from the time-series data of the reaction coordinate, $\boldsymbol{\xi}\left(t\right)$. The function $W\left(\boldsymbol{\zeta}\right)$ is also referred to as the **potential of mean force (PMF)**.

### Theoretical Background

Let ${\bf X}$ denote the configuration of the system and $U\left({\bf X}\right)$ its potential energy. The configurational integral $Z$ is given by

\begin{align}
Z =\int d{\bf X}\,\exp\left(-\beta U\left({\bf X}\right)\right)
\end{align}

where $\beta$ is the inverse temperature.

Similarly, the partition function corresponding to the state in which the system is trapped within the infinitesimal region $\boldsymbol{\zeta} - \boldsymbol{\zeta}+\Delta \boldsymbol{\zeta}$ in the reaction coordinate space ($\boldsymbol{\zeta}\left({\bf X}\right)$) is

\begin{align}
Z\left(\boldsymbol{\zeta}\right) =\Delta\boldsymbol{\zeta}\int d{\bf X}\,\delta\left(\boldsymbol{\zeta}-\boldsymbol{\xi}\left({\bf X}\right)\right)\exp\left(-\beta U\left({\bf X}\right)\right)
\end{align}

Defining the probability density corresponding to $\boldsymbol{\zeta}$ as

\begin{align}
\rho\left(\boldsymbol{\zeta}\right) = \Braket{\delta\left(\boldsymbol{\zeta} - \boldsymbol{\xi}\left({\bf X}\right)\right)} = \dfrac{1}{Z}\int d{\bf X}\,\delta\left(\boldsymbol{\zeta}-\boldsymbol{\xi}\left({\bf X}\right)\right)\exp\left(-\beta U\left({\bf X}\right)\right)
\end{align}

we obtain

\begin{align}
\dfrac{Z\left(\boldsymbol{\zeta}\right)}{Z} =\rho\left(\boldsymbol{\zeta}\right)\Delta\boldsymbol{\zeta}
\end{align}

Therefore, the free-energy difference $\Delta F\left(\boldsymbol{\zeta}\right)$ between the unrestricted state and the state confined to the region $\boldsymbol{\zeta}-\boldsymbol{\zeta} + \Delta \boldsymbol{\zeta}$ is expressed as

\begin{align}
\Delta F\left(\boldsymbol{\zeta}\right) =-\dfrac{1}{\beta}\log\dfrac{Z\left(\boldsymbol{\zeta}\right)}{Z}
 = -\dfrac{1}{\beta}\log \left(\rho\left(\boldsymbol{\zeta}\right)\Delta \boldsymbol{\zeta}\right)
\end{align}

The free energy relative to a reference point $\boldsymbol{\zeta}_{0}$ is

\begin{align}
W\left(\boldsymbol{\zeta}\right) =\Delta F\left(\boldsymbol{\zeta}\right)-\Delta F\left(\boldsymbol{\zeta}_{0}\right)\notag\\
 =-\dfrac{1}{\beta}\log\dfrac{\rho\left(\boldsymbol{\zeta}\right)}{\rho\left(\boldsymbol{\zeta}_{0}\right)}
 \label{pmf:pmf}
\end{align}

This quantity $W$ is called the **potential of mean force**. The choice of $\boldsymbol{\zeta}_{0}$ is arbitrary, but in most cases it is chosen such that the minimum value of $W\left(\boldsymbol{\zeta}\right)$ becomes zero.

By calculating $W\left(\boldsymbol{\zeta}\right)$, it becomes possible to discuss the change in thermodynamic stability as a function of $\boldsymbol{\zeta}$.

Equation \eqref{pmf:pmf} can also be rewritten by separating the $\boldsymbol{\zeta}$-dependent part and a constant $C$:

\begin{align}
W\left(\boldsymbol{\zeta}\right) =-\dfrac{1}{\beta}\log\int d{\bf X}\,\delta\left(\boldsymbol{\zeta}-\boldsymbol{\xi}\left({\bf X}\right)\right)\exp\left(-\beta U\left({\bf X}\right)\right)+C
\label{pmf:pmf2}
\end{align}

where

\begin{align}
C = \dfrac{1}{\beta}\log\int d{\bf X}\,\delta\left(\boldsymbol{\zeta}_{0}-\boldsymbol{\xi}\left({\bf X}\right)\right)\exp\left(-\beta U\left({\bf X}\right)\right)
\end{align}

### &option_param

* **``ndim``**
    * Default: None
    * Example: ``ndim = 2``

    Specifies the number of dimensions of the Collective Variable (CV).

    For example, ``ndim = 2`` calculates a two-dimensional PMF, while ``ndim = 3`` calculates a three-dimensional PMF.

----
* **``xyzcol``**
    * Default: ``1``
    * Example: ``xyzcol = 1 2``

    Specifies the column indices corresponding to each CV in the CV data file.

    The first column is ignored because it is treated as the time or step index.

    For example, if the data are stored as

    ```
    time  cv1  cv2  cv3
    ```

    then

    ```
    xyzcol = 1 2
    ```

    calculates a two-dimensional PMF using ``cv1`` and ``cv2``.

----
* **``ng3``**
    * Default: ``1 1 1``
    * Example: ``ng3 = 10 20``

    Specifies the number of grid points for each CV direction.

    When ``ndim = N``, the grid numbers for each dimension should be specified sequentially in ``ng3``.

----
* **``del``**
    * Default: ``1.0 1.0 1.0``
    * Example: ``del = 0.1d0 0.2d0``

    Specifies the grid spacing for each CV direction.

    For the $i$-th CV, if the grid spacing is denoted by $\Delta_i$, the distance between adjacent grid points is $\Delta_i$.

----
* **``origin``**
    * Default: ``0.0 0.0 0.0``
    * Example: ``origin = 0.0d0 1.0d0``

    Specifies the grid origin for each CV direction.

    For the $k$-th CV, let the origin be $x_k^{(0)}$ and the grid spacing be $\Delta_k$. Then the position of the $i$-th grid point is given by

    \begin{align}
    x_k^{(i)} = x_k^{(0)} + (i-1)\Delta_k
    \end{align}

----
* **``temperature``**
    * Default: ``300.0d0``
    * Example: ``temperature = 298.0d0``

    Specifies the temperature $T$ [K] used for the PMF calculation.

----
* **``use_radial``**
    * Default: ``.false.``
    * Example: ``use_radial = .true.``

    Specifies whether the CV contains a radial component.

    If ``.true.``, the distribution function for the CV component $x_k$ specified by ``radcol`` is calculated using the bin volume

    \begin{align}
    4\pi x_{k}^{2} \Delta_{k}^{2}
    \end{align}

----
* **``radcol``**
    * Default: N/A
    * Example: ``radcol = 1``

    Specifies which CV corresponds to the radial component.

    This parameter is effective only when ``use_radial = .true.``.

----
* **``box_system``**
    * Default: ``0.0 0.0 0.0``
    * Example: ``box_system = 50.0d0 50.0d0 50.0d0``

    Specifies the simulation box size.

    This parameter is used only when ``ndim = 3``.
    The box lengths in each direction should be specified in order.

----
* **``use_spline``**
    * Default: ``.false.``
    * Example: ``use_spline = .true.``

    Specifies whether spline interpolation is performed after the PMF calculation.

    If ``.true.``, spline interpolation is applied; otherwise, the original grid is used.

----
* **``spline_resolution``**
    * Default: ``4``
    * Example: ``spline_resolution = 8``

    Specifies the resolution used for spline interpolation.

    Larger values generate a finer interpolation grid.

----
* **``gen_script_mpl2d``**
    * Default: ``.false.``
    * Example: ``gen_script_mpl2d = .true.``

    Specifies whether a Python script for visualizing a two-dimensional PMF is generated.

    Currently, this option is supported only when ``ndim = 2``.

----
* **``skip_calc``**
    * Default: ``.false.``
    * Example: ``skip_calc = .true.``

    Specifies whether the PMF calculation is skipped.

    If ``.true.``, the PMF calculation is omitted and only auxiliary files are generated.
    If ``.false.``, the PMF calculation is performed normally.

    This option is convenient when recreating the Matplotlib (Python) script after modifying its settings.

### &matplotlib_param

Options for generating a Python (Matplotlib) script for visualizing a two-dimensional PMF.

These options are enabled when ``gen_script_mpl2d = .true.`` is specified in ``&option_param``.

* **``fpdf``**
    * Default: None
    * Example: ``fpdf = "pmf.pdf"``

    Specifies the name of the output PDF file.

----
* **``labels``**
    * Default: None
    * Example: ``labels = "$x$" "$y$" "PMF"``

    Specifies the axis labels displayed in the plot.

    When ``ndim = 2``,

    ```
    labels = "xlabel" "ylabel" "zlabel"
    ```

    specifies the labels for the $x$-axis, the $y$-axis, and the color bar corresponding to the PMF values, respectively.

    LaTeX-style mathematical expressions can be used.

----
* **``ranges``**
    * Default: None
    * Example: ``ranges = -10.0 10.0 -5.0 5.0 0.0 10.0``

    Specifies the plotting range.

    When ``ndim = 2``,

    ```
    ranges = xmin xmax ymin ymax zmin zmax
    ```

    should be specified in this order.

    Here,

    * ``xmin``, ``xmax`` : Display range of the $x$-axis
    * ``ymin``, ``ymax`` : Display range of the $y$-axis
    * ``zmin``, ``zmax`` : Display range of the PMF values

----
* **``tics``**
    * Default: None
    * Example: ``tics = 1.0 1.0 0.5``

    Specifies the intervals of the axis ticks.

    When ``ndim = 2``,

    ```
    tics = xtic ytic ztic
    ```

## Restricted Radial Potential of Mean Force (``restricted_radial``, ``rrpmf``)

This analysis mode calculates the free energy profile along the radial direction under a condition defined by one or more reaction coordinates (collective variables).

Using this profile, the free energy change associated with the binding of a host molecule and a guest molecule (binding free energy) can also be evaluated.

### Theoretical Background

Let ${\bf X}$ denote the configuration of the system.
The intermolecular distance between the host and guest molecules and the reaction coordinates defining the binding state are denoted by $d\left({\bf X}\right)$ and $\boldsymbol{\xi}\left({\bf X}\right)$, respectively.

The potential of mean force along the intermolecular distance is defined as

\begin{align}
w\left(r\right) =-\dfrac{1}{\beta}\log\left[\int d{\bf X}\,\dfrac{\delta\left(r-d\left({\bf X}\right)\right)}{4\pi r^{2}}\exp\left(-\beta U\left({\bf X}\right)\right)\right]+C
\end{align}

Furthermore, let $\boldsymbol{\Upsilon}$ be the region in the reaction coordinate space corresponding to the bound state, and introduce the characteristic function $\Theta\left(\boldsymbol{\xi}\right)$ defined as

\begin{align}
\Theta_{\mathrm{B}}\left(\boldsymbol{\xi}\right) =\begin{cases}
1 & \boldsymbol{\xi}\in\boldsymbol{\Upsilon}\\
0 & \boldsymbol{\xi}\notin\boldsymbol{\Upsilon}
\end{cases}
\end{align}

Using this function, the PMF for configurations belonging to the bound state is expressed as

\begin{align}
w_{\mathrm{B}}\left(r\right) =-\dfrac{1}{\beta}\log\left[\int d{\bf X}\,\dfrac{\delta\left(r-d\left({\bf X}\right)\right)}{4\pi r^{2}}\Theta_{\mathrm{B}}\left(\boldsymbol{\xi}\left({\bf X}\right)\right)\exp\left(-\beta U\left({\bf X}\right)\right)\right]+C
\end{align}

The constant $C$ appearing in both $w\left(r\right)$ and $w_{\mathrm{B}}\left(r\right)$ is taken to be the same value, and is chosen such that the minimum value of $w_{\mathrm{B}}\left(r\right)$ becomes zero.

Since $w\left(r\right)$ becomes constant when $r$ is sufficiently large, let this constant be $\Delta w$, and define the region satisfying $w\left(r\right)=\Delta w$ as $r_{0}\leq r\leq r_{1}$.
The choice of $r_{0}$ and $r_{1}$ is arbitrary as long as this condition is satisfied.
This point will be discussed later.

The probability ratio between the unbound state U, where the two molecules are separated by $r_{0}\leq r \leq r_{1}$, and the bound state B is

\begin{align}
\dfrac{{\displaystyle \int_{\boldsymbol{\Upsilon}}dr\,\exp\left(-\beta w_{\mathrm{B}}\left(r\right)\right)}}{{\displaystyle \int_{r_{0}}^{r_{1}}dr\,\exp\left(-\beta w\left(r\right)\right)}}
\end{align}

Therefore, the free energy change $\Delta G_{\mathrm{PMF}}$ associated with the transition from U to B is

\begin{align}
\Delta G_{\mathrm{PMF}} =-\dfrac{1}{\beta}\log\dfrac{{\displaystyle \int_{\boldsymbol{\Upsilon}}dr\,4\pi r^{2}\exp\left(-\beta w_{\mathrm{B}}\left(r\right)\right)}}{{\displaystyle \int_{r_{0}}^{r_{1}}dr\,4\pi r^{2}\exp\left(-\beta w\left(r\right)\right)}}
\end{align}

Since $w\left(r\right)=\Delta w$ within the range $r_{0}\leq r\leq r_{1}$, this equation can be rewritten as

\begin{align}
\Delta G_{\mathrm{PMF}} =-\Delta w-\dfrac{1}{\beta}\log\dfrac{V_{\mathrm{B}}}{V_{\mathrm{U}}}
\label{pmf:dGpmf}
\end{align}

where

\begin{align}
V_{\mathrm{B}} =\int_{\boldsymbol{\Upsilon}}dr\,4\pi r^{2}\exp\left(-\beta w_{\mathrm{B}}\left(r\right)\right)\\
V_{\mathrm{U}} =\int_{r_{0}}^{r_{1}}dr\,4\pi r^{2}
\end{align}

The second term in Eq. \eqref{pmf:dGpmf} represents the free energy change associated with the transition of the guest molecule from a concentration of $1/V_{\mathrm{U}}$ to a concentration of $1/V_{\mathrm{B}}$.

The experimentally measured binding free energy is the free energy change associated with binding a guest molecule at the standard-state concentration (typically $c^{\circ} = 1~\mathrm{mol~L^{-1}}$), and is therefore different from $\Delta G_{\mathrm{PMF}}$.

Let $V^{\circ}$ be the volume corresponding to a concentration of $c^{\circ}$ for a single guest molecule.
Then $V^{\circ} = 1661$ Angstrom$^{3}$.

The free energy change $\Delta G^{\circ}_{\mathrm{corr}}$ associated with the transition from concentration $c^{\circ}=1/V^{\circ}$ to $1/V_{\mathrm{U}}$ is

\begin{align}
\Delta G^{\circ}_{\mathrm{corr}} = - \dfrac{1}{\beta} \log \dfrac{V_{\mathrm{U}}}{V^{\circ}}
\label{pmf:dGcorr}
\end{align}

Thus, by decomposing the transition from $1/V^{\circ}$ to $1/V_{\mathrm{B}}$ into the two steps,

\begin{align}
1/V^{\circ} \to 1/V_{\mathrm{U}}
\end{align}

and

\begin{align}
1/V_{\mathrm{U}} \to 1/V_{\mathrm{B}}
\end{align}

we obtain

\begin{align}
\Delta G^{\circ} = \Delta G_{\mathrm{PMF}} + \Delta G_{\mathrm{corr}}^{\circ}
\end{align}

Substituting Eqs. \eqref{pmf:dGpmf} and \eqref{pmf:dGcorr} into the above equation yields

\begin{align}
\Delta G^{\circ} = - \Delta w - \dfrac{1}{\beta} \log \dfrac{V_{\mathrm{B}}}{V^{\circ}}
\end{align}

Since this expression does not contain $V_{\mathrm{U}}$, the value of $\Delta G^{\circ}$ is independent of the choice of $r_{0}$ and $r_{1}$.

----
* **``xsep``**
    * Default: ``0.0d0``
    * Example: ``xsep = 1.0d0``

    Specifies the boundary positions between regions with different moving-average widths.

    This parameter is not used when ``nregion = 1``, in which case ``xsep = 0.0d0`` should be specified.

    For example, when dividing the domain into $N$ regions, $N-1$ boundary values must be specified.

----
* **``npoint``**
    * Default: None
    * Example: ``npoint = 3 5``

    Specifies the number of data points used for the moving average in each region.

    An odd number must be specified.

    In the above example,

    * Region 1: 3-point moving average
    * Region 2: 5-point moving average

    are applied.

----
* **``include_zero``**
    * Default: ``.true.``
    * Example: ``include_zero = .false.``

    Specifies whether to preserve the value at the origin after smoothing.

    If ``.true.``, the value at the origin is kept unchanged from the input data.

### Output File Format

* ``*XXXX.dat``
    * Column 1: $x$ coordinate
    * Column 2: Function value

    A smoothed data file is generated for each input file specified by ``flist_cv`` or related options.

### Example

The following example smooths a one-dimensional function $f\left(x\right)$ stored in ``input.dat`` (grid origin ``0``, grid spacing ``0.1``).

The domain is divided into two regions, $x\leq 1.0$ and $x>1.0$, and 3-point and 5-point moving averages are applied to the first and second regions, respectively.

```
#!/usr/bin/env bash

cat << EOF > run.inp
&input_param
 fcv      = "input.dat"
/
&output_param
 fhead = "out"
/
&option_param
  dx           = 0.1d0
  xsta         = 0.0d0
  xsep         = 1.0d0
  nregion      = 2
  npoint       = 3 5
  include_zero = .true.
/
EOF

fanatra moving_average run.inp
```

Executing this command generates the smoothed function data file ``out0001.dat``.
## Average Function (``average_function``, ``avef``)

In this analysis mode, one-dimensional functions stored in multiple input files are averaged.
For example, let $f_{i}\left(x\right)$ be the function stored in the $i$-th file and let $N$ be the total number of files.
The averaged function $f_{\mathrm{ave}}\left(x\right)$ is given by

\begin{align}
f_{\mathrm{ave}}\left(x\right) = \dfrac{1}{N}\sum_{i=1}^{N}f_{i}\left(x\right),
\end{align}

### Parameter List

**Namelist**      | **Parameter**     | **Description**
:-----------------|:------------------|:-------------------------------------------------------------
`&input_param`    | `flist_cv`        | File containing a list of function data files
`&output_param`   | `fhead`           | Output file header
`&option_param`   | `use_bootstrap`   | Enable bootstrap analysis
`&option_param`   | `xsta`            | Starting position of the grid
`&option_param`   | `dx`              | Grid spacing
`&bootopt_param`  | `duplicate`       | Whether duplicate sampling is allowed
`&bootopt_param`  | `iseed`           | Random seed
`&bootopt_param`  | `nsample`         | Number of samples drawn in each trial
`&bootopt_param`  | `ntrial`          | Number of bootstrap trials

### ``&option_param``

* **``use_bootstrap``**
    * Default: ``.false.``
    * Example: ``use_bootstrap = .true.``

    Specifies whether to perform error estimation using the bootstrap method.

    If ``.true.``, bootstrap analysis is carried out according to the settings in ``&bootopt_param``.

----
* **``xsta``**
    * Default: ``0.0d0``
    * Example: ``xsta = 0.0d0``

    Specifies the starting position of the grid.

    If the position of the $i$-th grid point is denoted by $x_i$,

    \begin{align}
    x_i = x_0 + (i-1)\Delta x
    \end{align} 

    where $x_0$ is the grid origin.

----
* **``dx``**
    * Default: ``0.1d0``
    * Example: ``dx = 0.1d0``

    Specifies the grid spacing of the function data.

### ``&bootopt_param``

This section defines the settings for bootstrap error analysis.

It is effective only when ``use_bootstrap = .true.``.

* **``iseed``**
    * Default: Automatically generated
    * Example: ``iseed = 3141592``

    Specifies the random seed used for bootstrap analysis.

    If ``iseed <= 0``, the program automatically generates a seed.

----
* **``nsample``**
    * Default: None
    * Example: ``nsample = 1000``

    Specifies the number of samples drawn in each bootstrap trial.

----
* **``duplicate``**
    * Default: ``.true.``
    * Example: ``duplicate = .false.``

    Specifies whether duplicate sampling is allowed during bootstrap sampling.

----
* **``ntrial``**
    * Default: None
    * Example: ``ntrial = 100``

    Specifies the number of bootstrap trials.

    Increasing the number of trials improves the accuracy of the estimated statistical error, but also increases the computational cost.

### Output File Format

* ``*.ave``
    * Column 1: $x$ coordinate
    * Column 2: Function value
    * Column 3: Standard error (when ``use_bootstrap = .true.``)

### Example

Suppose that one-dimensional functions $f\left(x\right)$ (grid origin ``0``, grid spacing ``0.1``) are stored in multiple files, and the list of these files is stored in ``list``.
The following example calculates the average function and its standard error using the bootstrap method.

```
#!/usr/bin/env bash

cat << EOF > run.inp
&input_param
 flist_cv = "list"
/
&output_param
 fhead         = "out"
/
&option_param
 use_bootstrap =  .false.
 xsta          =  0.0d0  
 dx            =  0.1d0  
/

&bootopt_param
 duplicate     = .true.   
 iseed         =  3141592 
 nsample       =  1000    
 ntrial        =  100     
/
EOF

fanatra average_function run.inp 
```

Executing this command generates the averaged function data file ``out.ave``.
